内存管理

内存寻址

页

线性地址（虚拟地址）被分成以固定长度为单位的组，称为页（page）；页内部连续的线性地址被映射到连续的物理地址；页既指一组线性地址，又指包含在这组地址中的数据。

RAM分成固定长度的页框（page frame），又称物理页，是主存的一部分。

把线性地址映射到物理地址的数据结构称为页表（page table），存放在主存中。

页描述符

内核必须对页框进行管理，以区分页框是否空闲，同时区分那些非空闲的页框中哪些页框包含的是属于进程的页，哪些页框包含的是内核代码或内核数据。内核使用结构page记录页框的状态信息，该数据结构也称为页描述符。

地址空间

32位系统中，进程共拥有 0x00000000～0xffffffff 共 4G 的线性地址空间。

  • 0x00000000 ~ 0xbfffffff 共 3G 的进程地址空间，进程运行于用户态／内核态时可以寻址；
  • 0xc0000000 ~ 0xffffffff 共 1G 的内核地址空间，进程运行于内核态时才可寻址。

内核地址空间

内存管理区

内核地址空间从 0xc0000000 开始的线性地址线性地映射到 0x0 开始的物理内存地址。如果直接建立线性映射，内核只能访问前 1G 的物理内存，因而必须建立某种机制，使得更多的物理内存映射到内核地址空间。

x86结构将内核地址空间划分为三个内核管理区：

管理区 地址范围
ZONE_DMA 前16MB
ZONE_NORMAL 16MB～896MB
ZONE_HIGHMEM 896MB～1024MB

ZONE_HIGHMEM即为高端内存，内核将896MB之后的物理内存映射至高端内存的128M线性地址空间。

上图为内核地址的空间划分，PAGE_OFFSET 即为 0xc0000000，high_memory 之后的为高端内存。

地址 区域 描述
前896MB Physical Memory Mapping 物理内存直接映射
VMALLOC_START ~ VMALLOC_END vmalloc area 非连续内存分配
PKMAP_BASE ~ FIXADDR START Persistent Kernel Mappings 永久内核映射
FIXADDR START ~ 4G Fix-mapped linear addresses 固定映射的线性地址，临时内核映射

分配内存页，返回页的线性地址

分配以字节为单位的内存

gfp_mask标志

标志分为三类：行为修饰符、区修饰符、类型标志
行为修饰符表示内核应当如何分配所需的内存，区修饰符表示从哪个内存管理区分配内存，类型标志组合了行为修饰符和区修饰符。

区修饰符表示从哪个内存管理区分配内存，默认为ZONE_NORMAL

区修饰符 描述
__GFP_DMA 从ZONE_DMA分配
__GFP_HIGHMEM 从ZONE_HIGHMEM或ZONE_NORMAL分配
类型标志 描述
GFP_ATOMIC 原子地分配内存，不会睡眠，用于中断处理程序、下半部、持自旋锁的进程上下文
GFP_KERNEL 常规分配方式，可能睡眠，用于睡眠安全的进程上下文中
GFP_DMA 从ZONE_DMA区分配内存，用于获取DMA缓冲区的设备驱动程序

高端内存映射

调用alloc_pages()可以获得高端内存的页描述符，但高端内存的页框并不具有线性地址，必须将高端内存映射至内核地址空间。

  • 永久内核映射
  • 临时内核映射
  • 非连续内存分配

永久内核映射

永久内核映射允许内核建立高端内存页框到内核地址空间的长期映射；允许永久内核映射的数量是有限的。
这种映射可以休眠，只能用于进程上下文中。

调用kmap()建立永久内核映射，并返回该高端内存页框的线性地址；
调用kunmap()解除映射。

临时内核映射

固定映射的线性地址基本上是一种类似于0xffffc000这样的常量线性地址，其对应的物理地址不必等于线性地址减去0xc0000000，而是可以以任意方式建立。因此，每个固定映射的线性地址都映射一个物理内存的页框，并且可以映射任何物理地址。

临时内核映射正是借助这些“窗口”，允许内核通过保留的页表项原子地将高端内存页框映射至内核地址空间，
这种映射是非阻塞的，因而可以用于中断处理程序或可延迟函数的内部。

调用kmap_atomic()建立临时内核映射，并返回该高端内存页框的线性地址；
调用kunmap_atomic()解除映射；
参数type描述了临时内核映射的目的。

非连续内存分配

非连续内存分配允许连续的线性地址映射至非连续的物理地址，但需要修改页表。

  1. 寻找一个新的连续线性地址空间
  2. 依次分配一组非连续的页框
  3. 为线性地址空间和非连续页框建立映射关系，即修改内核页表

调用vmalloc()建立非连续内存分配，并返回该高端内存页框的线性地址；
调用vfree()解除映射；

vmalloc()分配物理地址不连续的内存，因而不能用于设备驱动程序，因为分配的线性地址只有经过内存处理单元MMU才能转换为非连续的物理地址，而设备通常不经过MMU直接访问内存。

slab

slab层缓存频繁分配和释放的数据结构，维护一个空闲链表，当需要分配数据结构时，从空闲链表获得一个空闲的数据结构；同时释放的数据结构返回空闲链表。

slab层为不同的对象（数据结构）维护一个高速缓存组，每个高速缓存包含多个slab，每个slab由一个或多个连续的物理页构成，并包含对象成员（缓存的数据结构）。

栈上的静态分配

以上介绍的分配方法都是内存的动态分配，在内核程序运行过程中动态地申请内存；此外还可以在栈上静态地分配内存，但是内核栈大小固定、空间较小，因而对大块内存空间的动态分配可以有效节省栈资源，防止栈溢出。

总结

  1. 分配连续的物理页，则调用alloc_pages()、__get_free_pages()、kmalloc()
  2. 分配高端内存页框，则调用alloc_pages()获得页描述符，之后调用kmap()将高端内存映射至内核地址空间
  3. 分配虚拟地址连续的页，不需要物理地址连续，则调用vmalloc()
  4. 频繁分配和释放大的数据结构，则调用slab高速缓存

进程地址空间

Linux系统采用虚拟内存技术，系统中的所有进程之间以虚拟方式共享内存，每个进程都可以访问所有的物理内存，其地址空间也可以远远大于系统的物理内存。

进程的地址空间由所有进程有权访问的线性地址区间构成，这种区间称为线性区（memory region），进程可以通过内核给自己的地址空间动态添加或减少线性区。

内存描述符

内核使用内核描述符结构体 mm_struct 表示进程的地址空间，该结构体包含了进程地址空间相关的所有信息。

线性区

内核使用结构体 vm_area_struct 描述线性区，该结构体描述了指定地址空间内的一个独立内存区间。内核将每个内存区间作为一个单独的内存对象管理，每个内存区间都拥有一致的属性，如访问权限等。
进程所拥有的线性区不重叠，对应于地址空间的唯一内存区间。分配新线性区时，内核尽力将新分配的线性区与紧邻的现有线性区进行合并。如果两个相邻线性区的访问权限相匹配，就能将它们合并。

内存映射

一个线性区可以和磁盘文件系统的普通文件的某一部分或块设备文件相关联，内核对线性区中页内的某一字节的访问将转换成对文件中相应字节的操作，这样的技术称为内存映射（memory mapping）。

出于安全性的考虑，用户进程无法直接访问设备内存。在设备驱动程序中实现内存映射，将设备内存的物理地址映射到进程地址空间，用户进程便可以直接访问设备内存，提高程序的吞吐量。如PCI设备驱动程序中，PCI设备将控制寄存器的物理地址映射到进程地址空间，应用程序可以通过访问内存地址直接访问寄存器，而不是频繁调用 ioctl 方法来获得配置信息。

文件内存映射依赖于请求调页机制。建立内存映射的过程实际上就是建立页表，使一块线性区 VMA 映射到一段物理内存的物理地址。一个新建立的内存映射是一个不包含任何页的线性区，当进程引用线性区中的一个地址时，缺页异常发生，缺页异常中断处理程序通过映射关系将磁盘上的文件或设备文件读取到物理内存中。
